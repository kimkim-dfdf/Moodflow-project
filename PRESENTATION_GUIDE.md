# MoodFlow 프레젠테이션 설명 가이드

## 1. 프로젝트 한 줄 요약
"사용자의 감정에 따라 맞춤형 할 일을 추천해주는 웹 앱입니다"

---

## 2. 기술 스택 설명 (면접관이 물어볼 때)

### Frontend (React)
**"화면에 보이는 부분을 만들었습니다"**
- 버튼 클릭하면 반응하는 것
- 데이터를 예쁘게 보여주는 것
- 페이지 이동하는 것

### Backend (Flask/Python)
**"데이터를 처리하는 부분입니다"**
- 로그인 확인
- 추천 알고리즘 계산
- 데이터베이스와 통신

### Database (PostgreSQL)
**"데이터를 저장하는 곳입니다"**
- 사용자 정보
- 할 일 목록
- 감정 기록

---

## 3. 핵심 알고리즘 설명 (가장 중요!)

### Weighted Scoring Algorithm (가중치 점수 알고리즘)
```
"각 할 일에 점수를 매겨서 높은 순서대로 추천합니다"

점수 = 카테고리 점수(57%) + 우선순위 점수(43%)
```

**예시로 설명하면:**
- 사용자가 "Happy" 감정을 선택
- Happy일 때는 Work 카테고리가 높은 점수 (에너지가 넘치니까 일하기 좋음)
- High 우선순위 작업이 높은 점수
- 이 두 가지를 합쳐서 총점을 계산

**왜 이렇게 했나요?**
"기분이 좋을 때는 어려운 일을 할 에너지가 있고,
피곤할 때는 쉬운 일을 하는 게 효율적이기 때문입니다"

### Tag AND Filtering (태그 필터링)
```
"여러 태그를 선택하면, 모든 태그를 가진 책만 보여줍니다"
```

**예시:**
- "Hopeful" + "Comforting" 태그 선택
- 두 태그를 모두 가진 책만 결과에 표시
- OR이 아닌 AND 로직 사용

## 4. 각 페이지 설명

### Dashboard (대시보드)
"메인 화면입니다. 감정을 선택하면 그에 맞는 할 일을 추천해줍니다"

### Tasks (할 일)
"추천받은 할 일들을 관리하는 곳입니다. 완료하면 체크할 수 있습니다"

### Calendar (캘린더)
"날짜별로 기록한 감정을 볼 수 있습니다"

### Books (책)
"기분에 맞는 책을 태그로 필터링해서 찾을 수 있습니다"

### Profile (프로필)
"사용자 설정을 관리하는 곳입니다"

## 5. 예상 질문과 답변

### Q: 왜 React를 선택했나요?
"컴포넌트 기반이라 코드 재사용이 쉽고, 
상태 관리가 편리해서 사용자 인터랙션이 많은 앱에 적합합니다"

### Q: 왜 Flask를 선택했나요?
"Python 기반이라 배우기 쉽고, 
가벼워서 빠르게 개발할 수 있습니다"

### Q: 알고리즘은 어떻게 작동하나요?
"두 가지 요소에 가중치를 줘서 점수를 계산합니다:
1. 감정과 카테고리 매칭 (57%)
2. 우선순위 (43%)
가장 높은 점수의 할 일을 위에 보여줍니다"

### Q: 데이터베이스 구조는?
"7개의 테이블이 있습니다:
- users (사용자)
- tasks (할 일)
- emotions (감정 종류)
- emotion_history (감정 기록)
- music_recommendations (음악)
- book_recommendations (책)
- book_tags (책 태그)"

### Q: 어려웠던 점은?
"감정에 따른 추천 로직을 설계하는 것이 가장 어려웠습니다.
어떤 감정일 때 어떤 카테고리가 적합한지 연구가 필요했습니다"

## 6. 데모 시나리오 (발표할 때)

1. **로그인** - "먼저 로그인합니다"
2. **감정 선택** - "오늘 기분이 피곤해서 Tired를 선택합니다"
3. **추천 확인** - "보시면 가벼운 작업들이 추천됩니다"
4. **할 일 추가** - "이 추천을 클릭해서 할 일에 추가합니다"
5. **완료 체크** - "완료하면 체크합니다"
6. **책 필터** - "책 페이지에서 태그로 필터링해봅니다"

## 7. 기술 용어 쉬운 설명

| 용어 | 쉬운 설명 |
|------|----------|
| API | 프론트엔드와 백엔드가 대화하는 방법 |
| REST API | 정해진 규칙으로 데이터를 주고받는 것 |
| Component | 재사용 가능한 UI 조각 (레고 블록 같은 것) |
| State | 현재 화면에 표시할 데이터 |
| Database | 데이터를 영구 저장하는 곳 |
| Query | 데이터베이스에서 데이터를 가져오는 명령 |
| Authentication | 로그인/로그아웃 처리 |
| Weighted Score | 여러 요소에 중요도를 다르게 줘서 계산 |
